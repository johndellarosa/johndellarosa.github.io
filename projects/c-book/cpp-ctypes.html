While ctypes is commonly used to call C functions directly, it can also be used to interact with C++ libraries. This chapter will explore how to use ctypes to work with C++ code, focusing on handling C++ classes, constructors, and destructors from Python.

Basic Compatibility

C++ ABIs are not stable across compilers or versions, and C++ adds features (overloading, templates, exceptions, name mangling) that don't map directly to ctypes. By writing a C wrapper around the C++ code, we can expose a C interface that ctypes can easily work with while still having the features of C++. This involves using `extern "C"` to prevent name mangling and ensure that the function names are compatible with C. 

                    #include <iostream>
                    using namespace std;

                    class MyClass {
                    public:
                        MyClass() {
                            cout << "MyClass constructor called!"  << endl;
                        }

                        ~MyClass() {
                            cout << "MyClass destructor called!" << endl;
                        }

                        void greet(const char* name) {
                            cout << "Hello, " << name  << "!"  << endl;
                        }
                    };

                    extern "C" {
                        MyClass* MyClass_new() {
                            return new MyClass();
                        }

                        void MyClass_delete(MyClass* obj) {
                            delete obj;
                        }

                        void MyClass_greet(MyClass* obj, const char* name) {
                            obj->greet(name);
                        }
                    }
In this code, we define a simple C++ class MyClass with a constructor, destructor, and a greet method. The extern "C" block is crucial because it tells the compiler to use C-style linkage, which is necessary for ctypes to interact with the functions without name mangling. See the C-API page for more about that.

Compiling

We can use g++ to compile the C++ code into a shared library. The command is similar to compiling C code, but we use g++ instead of gcc: 

                    g++ -shared -o my_cpp_functions.dll my_cpp_functions.cpp -fPIC

Macro


                    // api_export.h
                    #pragma once

                    #if defined(_WIN32)
                    #define API_EXPORT extern "C" __declspec(dllexport)
                    #elif defined(__GNUC__)
                    #define API_EXPORT extern "C" __attribute__((visibility("default")))
                    #else
                    #define API_EXPORT extern "C"
                    #endif
This macro defines API_EXPORT as an extern "C" function with the appropriate export attributes for different platforms. You can include this header in your C++ code to ensure that the functions are exported correctly. 

                    #pragma once
                    #include "api_export.h"

                    API_EXPORT MyClass* MyClass_new() {
                        return new MyClass();
                    }

                    API_EXPORT void MyClass_delete(MyClass* obj) {
                        delete obj;
                    }

                    API_EXPORT void MyClass_greet(MyClass* obj, const char* name) {
                        obj->greet(name);
                    }
