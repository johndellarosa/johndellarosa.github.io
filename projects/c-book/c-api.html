An alternative to ctypes is the Python C API, which allows you to write C extensions that can be imported directly into Python. This is more complex but offers better performance and integration with Python's memory management.

Anatomy of a C Extension

A C extension in Python is a shared library that can be loaded by the Python interpreter. It typically includes:
  • Header files that define the functions and types used in the extension.
  • Source files that implement the functions.
  • A setup script (usually named setup.py) that uses setuptools to build the extension.
  • A module definition file (usually named module.c) that defines the methods and initialization function for the extension.

Basic Example

The first thing that we will do is create a C file using the Python header. 


                #include <Python.h>

                // 1. C function exposed to Python
                static PyObject* helloworld(PyObject *self, PyObject *args) {
                    const char *name;
                    // 2. Parse Python arguments: expect a single string
                    if (!PyArg_ParseTuple(args, "s", &name)) {
                        return NULL; // Error in parsing arguments

                    }
                    // 3. C code: print greeting
                    printf("Hello, %s!\n", name);
                    // 4. Return None to Python
                    Py_RETURN_NONE;
                }

                // 5. Method Table: array lists functions exported to Python: name, C wrapper, calling convention, and docstring.
                static PyMethodDef Methods[] = {
                    {"hello", helloworld, METH_VARARGS, "Greet someone."},
                    {NULL, NULL, 0, NULL}
                };

                // 6. Module definition

                static struct PyModuleDef moduledef = {
                    PyModuleDef_HEAD_INIT,
                    "intro", // module name in python
                    "Basic API examples", // module docstring
                    -1, // no per-interpreter state
                    Methods
                };

                // 7. Module init function
                PyMODINIT_FUNC PyInit_intro(void) {
                    return PyModule_Create(&moduledef);
                }

                

                    from setuptools import setup, Extension

                    ext = Extension(
                        'intro',
                        sources=['hello.c'],
                    )

                    setup(
                        name='intro',
                        version='0.1',
                        ext_modules=[ext],
                    )
To compile this C extension, you would typically run the following command in the terminal: 

                    python setup.py build_ext --inplace
This command uses setuptools to build the C extension in place, meaning it will create a shared library file (e.g., intro.so on Linux or intro.pyd on Windows) in the current directory. 

                    import intro

                    # Call the C extension function
                    intro.hello("World") # prints "Hello World!"

Memory Management

In CPython, every object is represented by a PyObject (or a type derived from it). The PyObject struct always starts with a reference count.

Reference Counting

Reference counting is the primary memory management technique in CPython. Each object has a reference count that is incremented when a new reference to the object is created and decremented when a reference is deleted. When the reference count reaches zero, the object's memory is freed. There are two types of reference counts:
  • Strong Reference: A strong reference is defined in the Python C API glossary as "a reference to an object which is owned by the code holding the reference." The functions Py_IncRef and Py_DecRef will increase and release a strong reference to an object respectively.
  • Borrowed Reference: A borrowed reference does not increase the reference count. It is used when you want to temporarily use an object without taking ownership of it.
When working with the Python C API, it's crucial to manage reference counts correctly to avoid memory leaks or crashes. Example of a new reference: 

                    PyObject* obj = PyFloat_FromDouble(3.14); // Create a new reference to a float object
                    // Use obj
                    Py_DECREF(obj); // Decrement reference count
Example of a borrowed reference: 

                    PyObject* obj = PyList_GetItem(my_list, 0); // Get a borrowed reference to the first item in a list
                    Py_INCREF(item); // make it yours

                    // ... do work ...

                    Py_DECREF(item); // free when done
The C API will explain which type of reference a function returns.

Cycles

Cycles can occur when two or more objects reference each other, preventing their reference counts from reaching zero. To handle this, Python uses a cyclic garbage collector that periodically checks for cycles and breaks them.

Accessing Python Functions from C

You can call Python functions from C using the Python C API using the PyRun_SimpleString function. Additionally, you can use PyObject_CallObject to call specific Python functions.
